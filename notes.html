Getting Started

Download the compo.js or use it as a link to this GitHub repo.
Always put the <script> tag linking to it at the end of your <body> tag.


<script src="https://adir-sl.github.io/compojs/compo.js"></script>
OR 
<script src="compo.js"></script>

Basic Usage

All custom tags should be hyphenated eg. button-primary, custom-input, radoi-group etc. This library will take code like:

<button-primary>...</button-primary>
and replace it with the corresponding file named button-primary.html.

Also the CSS coming from the file named button-primary.html should be scoped to not affect other buttons.

Props

You can also add props to change some properties in the component you're loading. All props should be inside curly brackets eg. {prop}. In your host file:

<button-primary {label}="button-1">...</button-primary>
<button-primary {label}="button-2">...</button-primary>
<button-primary {label}="button-3">...</button-primary>

In your component file (named button-primary.html):

<!-- button-primary.html -->
<button>{label}</button>

This will result in 3 buttons with numbered labels, all loaded from the same component file (button-primary.html).

Can I use props for variable content? Yes, props are used kind of like variables but with a different syntax, for example you can use the code below and just get 3 different buttons, each one with a different label. 
<button-primary {label}="button-1">...</button-primary>
<button-primary {label}="button-2">...</button-primary>
<button-primary {label}="button-3">...</button-primary>
Is it possible to have a compo.js component inside another compo.js component? Not at the moment, this may come in future versions.
With clever architecture, this might not even be an issue.
Can I use compo.js with a CDN? Certainly, compo.js can be used directly from GitHub like so: 
<script src="https://adir-sl.github.io/compojs/compo.js"></script>
It can also be used locally like so: 
<script src="compo.js"></script>
More CDN options may come in the future.
Sometimes I can see the styles of wrong components That might happen if your server doesn't respond quick enough, could be due to physical distance, or a virtual one like a VPN.
A quick fix can be to manipulate the numbers in the compo.js file at the end of findTags() function. Just don't reverse the order by making the first number lower than the second one. 
...
    setTimeout(function(){ editStyles(); }, 250);
    setTimeout(function(){ replaceProps(); }, 10);
...
All my components are at the same place on the page That might happen if you work with absolute or fixed positioning on all of your components. Try to avoid it by adjusting your positioning to relative so your page will flow as expected, every component layed out one after the other.
